# The audio-store owns the playback-rate clamp; PlaybackEngine trusts its caller

`Math.max(0.25, Math.min(2, rate))` existed identically in both `audio-store.setPlaybackRate` and `html-audio.setPlaybackRate` — a real duplication with two candidate owners: the store (domain: what's a valid playback intent) or the engine (device: what the `<audio>` element itself will accept, which is far wider than `[0.25, 2]`). We chose the store: `[0.25, 2]` is a product decision about acceptable listening speeds, not a browser constraint, so `clampPlaybackRate` lives once as a pure function in `lib/playback-engine.ts`, and `PlaybackEngine.setPlaybackRate` assumes an already-valid value. A future reader who finds the engine's `setPlaybackRate` unclamped should not "fix" it by re-adding the clamp there — that reintroduces the duplication this decision removed. If a second `PlaybackEngine` adapter needs its own hard safety limit for a different reason (device-specific range), reopen this decision rather than silently duplicating the store's clamp.
